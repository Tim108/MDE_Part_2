Analysis of HamersApp

This page represents the analysis of the android application "HamersApp". A table of contens is shown on the left hand side with links to the section with differnt analysed elements of the application. This document contains several colors that indicate warnings or errors found in the code. Red indicates errors that were found with the structure of the android application. Orange indicates warnings that were encountered in the application. Warnings do not always mean that something is definitialy wrong, but might be wrong with the code. Green indicates that the analysed element was deemed correct.

Analysis per Presenter

The table below shows for each presenter to which models and views this presenter is connected. In a perfect scenario, a presenter is connected to a single view and model. It is also allowed for a presenter to connect to multiple views, this to reduce the amount of code that is used inside a single view. It is recommended for a presenter to connect to a single model, but connecting to multiple models is allowed.
